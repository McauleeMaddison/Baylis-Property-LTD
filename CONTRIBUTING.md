Contributing
============

Thank you for contributing to Baylis-Property-LTD. Please follow these guidelines:

- Open an issue to discuss major changes before you code them.
- Create feature branches named `feature/<short-desc>` or `fix/<short-desc>`.
- Run tests and linting before opening a PR.
- Write clear PR descriptions and reference any related issues.

Local workflow
--------------

```bash
git checkout -b feature/my-feature
# make changes
git commit -am "feat: add ..."
git push origin feature/my-feature
```

CI will run tests on PRs; do not merge until checks pass.
